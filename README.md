# RAG_abitura
RAG для ответов на вопросы абитуры

- тексты документов - предобработанные размеченные страницы сайта Волгатеха - загружаем в папку source; название файла документа - это его doc_id (не более 50 символов) (doc_id.md);
- результаты деления текста документа на чанки загружаем в папку chunks; название файла с описанием разделов и чанков должно начинаться с doc_id (doc_id_chunks.json);
- запросы, ответы на которые содержит документ, загружаем в папку CQs; название файла с описанием запросов должно начинаться с doc_id, в котором содержится ответ на запрос (doc_id_cqs.json)
- ttl-файлы с rdf - описанием документа, запросов (относящихся к документу), чанков (из документа) загружаем в папку rdf; название файла должно быть "doc_id.ttl"
- формируемые векторные хранилища запоминаем в папке vect_storage; название файла должно соответствовать типу хранящихся там сущностей (chunks.json, cqs.json)
- программные модули запоминаем в папке modules
- файлы тестовой базы запоминаем в папке test; файлы имеют формат json; названия прозвольные;


запуск системы через docker-compose up -d
